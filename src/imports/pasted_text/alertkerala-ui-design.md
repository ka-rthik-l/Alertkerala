Design a modern, professional emergency response web application UI called **AlertKerala**.

The application helps authorities broadcast geo-targeted missing person alerts to civilians in real time.

Style: clean government-tech interface, high trust design, minimalistic but powerful. Use colors like red (alerts), dark blue (authority), white background, and map-focused layout. The UI should feel similar to emergency response systems like disaster dashboards.

Create the following screens:

1. Landing Page

* Hero section with headline: "AlertKerala – Every Second Matters"
* Short explanation about real-time missing person alerts
* Illustration of a live map with alert pins spreading across a city
* Buttons: "Report Missing Person" and "Join Civilian Network"
* Section explaining how it works in 3 steps: Report → Broadcast → Sightings
* Stats section (alerts sent, civilians connected, response time)
* Footer with government-style credibility

2. Civilian Mobile Alert Screen

* Large red emergency alert banner
* Missing child photo
* Name, age, last seen location
* Time missing
* Map showing search radius
* Buttons:

  * "I Saw Them"
  * "Share Alert"
  * "Navigate to Area"
* Alert timer showing "Sent 12 seconds ago"

3. Report Missing Person Page

* Form with fields:

  * Upload photo
  * Name
  * Age
  * Last seen location
  * Time last seen
  * Description
  * Clothes description
  * Contact number
* Map selector to mark last seen location
* Radius selector slider (1km – 10km)
* Submit button: "Broadcast Alert"

4. Authority Command Dashboard

* Full screen live map of Kochi
* Red broadcast radius circle
* Sightings appearing as blinking pins
* Sidebar panel with:

  * Active alerts
  * Sightings list
  * Civilian responses
* Real-time updates indicator
* Alert timeline

5. Live Alert Broadcast Animation Screen

* Map of Kochi
* Ripple effect spreading from last seen location
* Notification cards popping up on civilians
* Text: "Alert sent to 1,243 civilians in 4 seconds"

6. Civilian Sighting Submission

* Button: "I Saw Them"
* Auto geolocation capture
* Optional photo upload
* Short message input
* Submit button
* Confirmation screen: "Sighting sent to authorities"

7. Real-Time Response Panel

* Command center style dashboard
* Metrics:

  * Civilians notified
  * Sightings received
  * Active responders
  * Estimated search radius
* Map heatmap of sightings

Design Requirements:

* Responsive layout
* Map-first interface
* Clear emergency colors
* Modern fonts
* Real-time indicators
* Mobile-first for civilians
* Dashboard-first for authorities

Include UI components like:

* Alert cards
* Map pins
* Radius overlays
* Live notification popups
* Status badges
* Real-time activity feed

The design should look like a real production emergency response platform suitable for government deployment.
